1
Synthetic Dataset Design
8,000 training / 3,200 test users. Three classes: Normal (50%), Compromised Account (25%), Admin Insider (25%). Designed with heavy feature overlap — all values stay within normal-user ranges on any single axis. 6 attack subtypes across 2 categories.
2
Feature Engineering
14 features across 4 layers: SIEM (3), EDR (3), NDR (3), XDR-only cross-layer (5). The 5 XDR-only features — such as role_based_access_score — require simultaneous access to identity + endpoint + network signals and cannot be computed by any single layer.
3
Model Architecture
Four XGBoost models trained in isolation — one per layer feature set. GridSearchCV over 54 hyperparameter combinations × 5 CV folds = 270 fits per model. Best XDR hyperparameters: n_estimators=200, max_depth=3, learning_rate=0.1, subsample=0.8.
4
6 Anti-Memorization Tests
Noise robustness, learning curves, held-out attack subtypes, cross-validation, train-test gap analysis, and permutation importance. Each test independently rules out a specific alternative explanation (overfitting, lucky split, algorithm bias, dataset size, feature noise).
5
Algorithm-Agnostic Validation
Models 01–05 use Random Forest (bagging). Model 06 uses XGBoost (boosting). Both confirm XDR's structural advantage (~0.199–0.212 F1 gap over best single layer). The advantage is architectural, not algorithm-specific.
6
Ablation Decomposition
Model 04 ablation: +0.116 F1 from combining single-layer features into one model (feature unification), +0.126 F1 from the 5 XDR-only cross-layer features. Cross-layer features are the primary driver.
